# Антидубль v2 — Виджет поиска и слияния дубликатов amoCRM

Виджет для amoCRM, который находит дубликаты контактов и объединяет их через нативный API v4 merge.

## Ключевые отличия от v1

| Аспект | v1 (старая) | v2 (новая) |
|--------|------------|------------|
| **API вызовы** | `self.crm_post()` + `$.ajax()` вручную | `AMOCRM.call()` — стандартный SDK |
| **Слияние** | Ручное копирование полей + удаление дублей | `POST /api/v4/contacts/merge` — нативный API |
| **Пагинация** | Ручная постраничная (page/limit) | `_links.next` из API v4 |
| **Retry** | Нет | Автоматический retry при 429/5xx |
| **Токен** | Требовал ручного ввода токена | Не нужен — работает от текущего пользователя |
| **Сравнение** | Только телефон + Telegram | Телефон + email + кастомное поле |
| **Стабильность** | Риск потери данных | Атомарное слияние через API amoCRM |

## Установка

1. Запаковать архив: `cd ~/widget-antidupl-v2 && zip -r /tmp/widget-antidupl-v2.zip . -x ".git/*"`
2. amoМаркет → Мои приложения → Добавить
3. Загрузить архив, сгенерировать код и ключ
4. Установить в аккаунте

## Использование

1. Открыть настройки виджета → выбрать поля для сравнения (телефон, email, кастомное поле)
2. В карточке контакта нажать «Найти дубликаты»
3. Выбрать группы для слияния → нажать «Слить»

## Технические детали

- **API:** REST API v4 (`/api/v4/contacts`, `/api/v4/contacts/merge`)
- **Авторизация:** `AMOCRM.call()` — права текущего пользователя
- **Rate limit:** Автоматический retry с экспоненциальной задержкой
- **Пагинация:** Полная загрузка через `_links.next` (250 записей за запрос)
